# Knowledge Distillation Pipeline

Pipeline for distilling the 8B teacher model (Llama 3.1 8B) into a 1B student model (Llama 3.2 1B) optimized for the navigation robot.

## Requirements

- Python 3.10+
- NVIDIA GPU with CUDA (RTX 4060 8GB recommended)
- `pip install unsloth` (automatically installs transformers, peft, trl, bitsandbytes)

## Execution flow

```
prepare_training_data.py → train_student.py → export_gguf.py
```

### Step 1: Prepare training data

```bash
python prepare_training_data.py
```

Transforms the raw responses from the 8B teacher into a clean dataset ready for fine-tuning. The process has 4 stages:

**1. Reading teacher results**

Reads the 4 `*_results.jsonl` files generated by `dataset_creator.py` in the previous step. Each line contains the input sent to the teacher, its response, and the validation result.

**2. Quality filtering**

Discards entries that are not suitable for training the student:
- Entries with no response (`response` empty or null)
- Entries where validation failed (`validation.valid == false`) — these are responses where the 8B teacher made a mistake (wrong result_type, wrong command, incorrect landmarks, etc.)
- Responses with malformed JSON

Only VALID responses are included in the training dataset. If the teacher had 53% accuracy, only that 53% is used — the student only learns from correct examples.

**3. Conversion to chat format**

Each validated entry is converted to the message format expected by HuggingFace's SFTTrainer:

```json
{"messages": [
  {"role": "system", "content": "<system prompt extracted from the ModelFile>"},
  {"role": "user", "content": "{\"version\":\"1.0\",\"request_type\":\"from_user\",\"user_message\":\"Take me to the kitchen\"}"},
  {"role": "assistant", "content": "{\"version\":\"1.0\",\"result_type\":\"interpreted_explicit_command\",\"assistant_message\":\"I'll take you to the kitchen.\",\"command\":{\"name\":\"move_to\",\"parameters\":{\"landmarks_to_visit\":[\"kitchen\"]}},\"follow_up\":null}"}
]}
```

- **system**: the full system prompt from the ModelFile (classification rules, format, etc.)
- **user**: the exact input JSON received by the teacher
- **assistant**: the teacher's JSON response, compacted (no pretty-print to save tokens)

**4. Train/eval split**

Randomly shuffles entries (with a fixed seed for reproducibility) and splits them 90/10:
- `train.jsonl` (90%) — used to adjust the student's weights
- `eval.jsonl` (10%) — used to measure loss on unseen data during training (detects overfitting)

**Input:** `dataset/{explicit,implicit,natural_language,invalids}/*_results.jsonl`
**Output:** `dataset/training/train.jsonl` + `eval.jsonl`

---

### Step 2: Train student model

Fine-tunes the student model using Unsloth + QLoRA + SFTTrainer.

Train with default parameters (3 epochs, batch 4, lr 2e-5):
```bash
python train_student.py --model-name distilled-1b-robot-router
```

Train with more epochs (more passes over the data, may improve accuracy but risks overfitting):
```bash
python train_student.py --model-name distilled-1b-robot-router --epochs 5
```

Verify that data and model load correctly without launching training:
```bash
python train_student.py --model-name distilled-1b-robot-router --dry-run
```

**Input:** `dataset/training/train.jsonl` + `eval.jsonl`
**Output:** `models/<model-name>/lora/` + `models/<model-name>/merged/`

| Argument | Default | Description |
|---|---|---|
| `--model-name` | (required) | Output directory name under `models/` |
| `--epochs` | 3 | Number of training epochs |
| `--batch-size` | 4 | Batch size per GPU |
| `--lr` | 2e-5 | Learning rate |
| `--max-seq-length` | 512 | Maximum sequence length in tokens |
| `--lora-r` | 16 | LoRA adapter rank |
| `--lora-alpha` | 32 | LoRA alpha (scales the effect of the adapters) |
| `--dry-run` | false | Load model and data without training |

### Step 3: Export to GGUF

Export with default quantization (Q4_K_M, ~0.6 GB):
```bash
python export_gguf.py --model-name distilled-1b-robot-router
```

Export with 8-bit quantization (higher quality, ~1.2 GB):
```bash
python export_gguf.py --model-name distilled-1b-robot-router -q q8_0
```

Converts the merged FP16 model to a quantized GGUF format for deployment on the N100.

**Input:** `models/<model-name>/merged/`
**Output:** `models/<model-name>/gguf/*.gguf`

| Argument | Default | Description |
|---|---|---|
| `--model-name` | (required) | Model directory name |
| `--quantization` / `-q` | q4_k_m | GGUF quantization type |

## Modules

### `utils.py`

Global variables and shared functions across the 3 scripts.

| Variable | Description |
|---|---|
| `CONFIG` | Dictionary with the configuration from `config.yaml` |
| `DATASET_DIR` | Path to the datasets directory |
| `MODELS_DIR` | Path to the model output directory |
| `SEED` | Reproducibility seed |
| `STUDENT_MODEL_ID` | Student model ID on HuggingFace |
| `TRAINING_CFG` | `training` section from `config.yaml` |
| `RESULT_FILES` | List of paths to the teacher's `*_results.jsonl` files |
| `TRAIN_FILE` / `EVAL_FILE` | Paths to the training datasets |

| Function | Description |
|---|---|
| `model_output_dir(name)` | Returns `MODELS_DIR / name` |
| `extract_system_prompt()` | Extracts the system prompt from the Ollama ModelFile |
| `load_jsonl(path)` | Loads a JSONL file into a list of dicts |

### `train_student.py` — internal functions

| Function | Description |
|---|---|
| `__parse_args()` | Parses CLI arguments with defaults from `config.yaml` |
| `__load_model(args)` | Loads the student model in 4-bit and adds LoRA adapters |
| `__load_datasets()` | Loads `train.jsonl` and `eval.jsonl` as HuggingFace Datasets |
| `__dry_run(train_dataset)` | Shows a dataset example and exits without training |
| `__train(...)` | Configures SFTTrainer and runs training |
| `__save_model(...)` | Saves LoRA adapters and merged FP16 model |

## Output structure

```
models/
  distilled-1b-robot-router/
    checkpoints/          # Intermediate checkpoints (auto-cleaned, max 2)
    lora/                 # LoRA adapters (~80 MB)
    merged/               # Full FP16 model (~2.5 GB)
    gguf/                 # Quantized GGUF model (~0.6 GB)
```
